Användning av SCSS

Det här var första gången jag använde SASS, men jag hade sett det förut. Jag förstår att CSS-processorer uppstod för att fylla hålen där standard CSS saknar funktioner, men idag finns saker som variabler, pseudo-element och fler selektorer inbyggda i standard CSS. Att använda SASS för detta lilla projekt kändes ganska onödigt, men målet var ju att testa på det. SASS styrkor kommer säkert fram i större projekt.

Jag använde den nya syntaxen för SCSS, så det liknade vanlig CSS och blev inte förvirrande. Tack vare Vite så krävdes ingen konfiguration och allt gick smidigt. Att använda partials och @use liknade arbetet med javascript och var lätt och logiskt att förstå.

Funktionerna jag använde, @use och @extend, gick snabbt att testa och dokumentationen var tydlig. Jag använde inte särskilt mycket av vad SASS har att erbjuda, men @extend verkar väldigt praktiskt för större projekt där man har flera variationer av element och komponenter.

Även om SASS är ett kraftfullt verktyg som jag göra mer än vad jag använde för det här projektet så tror jag inte att jag kommer använda det mycket i framtiden. För små projekt kan jag nog klara mig med standard CSS, och det är oftast bättre att ha så få verktyg inblandade som möjligt. Om jag träffar på något större projekt som använder SASS har jag nu i alla fall en grundläggande förståelse för hur det fungerar.
